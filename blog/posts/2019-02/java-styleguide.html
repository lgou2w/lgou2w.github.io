Java & Kotlin 编程风格规范指南

前言

这份文档是我个人的编程风格规范的定义。主要侧重于我所普遍遵循的规则。

源文件基础

1.1 文件名

源文件以其最顶层的类名来命名，大小写敏感，文件扩展名为 .java。Kotlin 为 .kt

1.2 文件编码

源文件编码均为 UTF-8 格式

1.2.1 设置 Intellij IDEA 源文件编码

File - Other Settings - Settings for New Projects - Editor - File Encodings

  1. 设置 Global Encoding 和 Project Encoding 为 UTF-8
  2. 设置默认 Properties Files 为 UTF-8
  3. 并设置 BOM 为 with No BOM

1.3 文件缩进

源文件缩进均使用 Tab 键方式缩进，一个缩进为 4 个空格。

1.3.1 设置 Intellij IDEA 源文件缩进

File - Other Settings - Settings for New Projects - Editor - Code Style - Java | Kotlin - Tabs and Indents

  1. 不要打勾使用 Tab character 制表符，应使用空格
  2. 设置 Tab size 为 4
  3. 设置 Indent 为 4
  4. 设置 Continuation indent 为 8

Kotlin 同理。

源文件结构

一个源文件包含以下结构:

  1. 许可证或版权信息 (如果需要)
  2. 包 package 语句
  3. 导入 import 语句
  4. 一个顶级类

每个部分之间用空行隔开。

2.1 许可证或版权信息

如果一个文件包含许可证或版权信息，那么它应当被放在文件最前面。

2.2 包 package 语句

package 语句不换行，写在一行里。

2.3 导入 import 语句

import 语句不换行，每个 import 语句独立成行。

2.3.1 不要使用通配符

即，不要出现类似这样的 import 语句：import java.util.*;

2.3.1.1 设置 Intellij IDEA 不使用通配符

File - Other Settings - Settings for New Projects - Editor - Code Style - Java | Kotlin - Imports

  • Java Imports
  1. 设置 Use single class import 打勾。使用单类导入
  2. 设置 Use fully qualified class names 打勾。使用完全限定类名
  3. 设置 Class count to use import with '*' 为 1000。要使用 * 导入类的计数
  4. 设置 Names count to use static import with '*' 为 1000。要使用 * 静态导入的名称计数
  5. 最后将 Packages to Use Import with '*' 全部移除。使用 * 导入的包
  • Kotlin Imports
  1. 设置 Top-level Symbols 和 Java Statics and Enum Members 均为 Use single name import 单名称导入
  2. 最后将 Packages to Use Import with '*' 全部移除。使用 * 导入的包

命名规范

3.1 驼峰命名法

驼峰式命名法分大驼峰式命名法 UpperCamelCase 和小驼峰式命名法 lowerCamelCase。 有时，我们有不只一种合理的方式将一个英语词组转换成驼峰形式，如缩略语或不寻常的结构(例如 IPv6 或 iOS)。

示例：

散文 正确 错误
XML HTTP request XmlHttpRequest XMLHTTPRequest
new customer ID newCustomerId newCustomerID
inner stopwatch innerStopwatch innerStopWatch
supports IPv6 on iOS? supportsIpv6OnIos supportsIPv6OnIOS

3.2 标识符类型的规则

3.2.1 包名

包名全部小写，连续的单词只是简单地连接起来，不使用下划线。 示例：com.lgou2w.myapp

3.2.2 类名

类名都以 大驼峰法 风格编写。 示例：HashMap、MyApplicationSetting

3.2.3 方法名

方法名都以 小驼峰法 风格编写。通常是动词或动词短语。 示例：sayHello、isValid、hasGravity、canPlay

3.2.4 变量和参数名

变量和参数名都以 小驼峰法 风格编写。 示例：String myName、int entityId、Object targetObj

3.2.5 常量名

常量名命名模式为 CONSTANT_CASE，全部字母大写，用下划线分隔单词。 示例：final static String MY_NAME = "foo"

格式

4.1 花括号约束

左花括号应当在一行的结尾，而不是又一行的开始。

// 正确示例：
public static void main(String[] args) {
    if (expression) {
        // do this
    } else {
        ...
    }
}
// 错误示例：
public static void main(String[] args)
{
    if (expression)
    {
        // do this
    }
    else
    {
        ...
    }
}

4.2 关键字括号空格

在一些关键字的括号应当空出一个空格，方便其阅读。

// 正确示例：
if (expression) {
}
// 错误示例：没有空格，增加阅读困难
if(expression){
}

// 一些正确示例：
for (int i = 0; i < 10; i++) {
}
while (expression) { 
}
try {
} catch (Exception e) {
}

4.3 过长的连续表达式应该换行

当判断多个条件表达式应当其换行，而不是全部在一行。

// 正确示例：
if (expression1 ||
    expression2 ||
    expression3 ||
    expression...) {
    ...
}
// 错误示例：
if (expression1 || expression2 || expression3 || expression...) {
}
// 当然如果表达式不是特别长的话可以不换行，例如：
if (a == 1 || a == 2 || a == 3) {
}

4.4 不要使用多层次嵌套逻辑块

多层次嵌套逻辑块会大大影响源代码的阅读性，强烈不要这样做。

// 错误的多层次嵌套逻辑块示例：
public static void main(String[] args) {
    if (expression1) {
        if (expression2) {
            if (expression3) {
                // do this
            }
        }
    }
}
// 正确示例：
public static void main(String[] args) {
    if (!expression1)
        return;
    if (!expression2)
        return;
    if (!expression3)
        return;
    // do this
}

4.5 Kotlin 格式

4.5.1 类声明规范

类有太多的成员或参数应当按照下列规范，而不是全部一行写死。

class Son(
    val id: Int,
    val name: String
) : Father(id, name), // 继承父类
        Comparator,   // 实现接口
        Runnable {    // 实现接口
}

4.5.2 变量格式规范

变量名的冒号应该左右空格，方便阅读。

val name : String? = null // 正确
val name:String? = null // 错误
1
2

变量的 Getters、Setters 或修饰应当比变量多一个 Tab 缩进。

// 正确实例：
var name : String
    get() = "name"
    private set
// 错误实例
var name : String
get() = "name"
private set

4.5.3 函数格式规范

如果函数没有返回值类型，那么应当为空。

fun hi() { } // 正确
fun hi(): Unit { } // 错误
1
2

函数参数的变量名冒号右边应当空格，方便阅读。

fun sayHello(msg: String) // 正确
fun sayHello(msg:String) // 错误
1
2

函数返回类型的冒号右边应当空格，方便阅读。

fun calc(a: Int, b: Int): Int // 正确
fun calc(a: Int, b: Int):Int // 错误
1
2

单表达式函数如果过长应当换行并且两个 Tab 缩进

// 正确实例：
fun calc(a: Int, b: Int): Int = a + b // 没有太长，可以不换行
fun balabalabalabala很长的函数名(a: Int, b: Int): Int
        = a + b // 两个缩进
// 错误实例：
fun balabalabalabala很长的函数名(a: Int, b: Int): Int
= a + b // 没有缩进
